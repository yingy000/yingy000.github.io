CCT6589_Basic之AF Preformance Tuning(1)

工具 CCT 2018/10/08

1.了解AF

现在大多数手机的AF都是被动式AF，也就是利用Focus Value Curve（曲线方程）来对焦。

1.1.Focus Value的理解

一般情况下，Focus Value越大代表越Sharp，也就意味着Focus Pos越接近焦点。

特殊情况：

  • 若亮度太低，Gain太大，会导致Noise影响Focus Value。

  • 过于Smooth没有Edge，导致Noise影响Focus Value。

1.2.被动式AF的目标

找到Focus Value Curve，Focus Value最大的Focus Position。

  • 如何在最短的时间内找到Peak Focus Value Position(AF的速度)=>峰值点

  • 如何找到最准确的Peak Focus Value Position(AF的精准度)=>峰值点

注意：速度和精度这两个特性很难同时满足，必须要依靠Trade-Off(平衡性)来取得最佳平衡=>原因：如果要求搜寻速度要快，那么Focus的间隔必须要大，这样搜寻的精准度势必不会好，同理如果要求搜寻的精准度高，搜寻的间隔肯定要小，搜寻的速度肯定会慢下来，这两者其实是相互矛盾的。

1.3.被动式AF的动作

  • Single AF(SAF)：单按对焦键时，对中心的AF Window执行一次AF（永远对画面中心对焦）

  • Touch AF(TAF)：单按屏幕时，对屏幕触点的位置执行一次AF（对触摸的位置对焦）

Single AF(SAF)和Touch AF(TAF)使用同一组调试参数，其中包括AF搜索的范围、AF Table、AF的精度、AF的速度等，基本上表现都会相同。=>只是对焦的主体不相同

  • Multi AF(MAF)：单按对焦键时，对中心的五块AF Window执行一次AF（最终选取最近焦的Window来当做主要的对焦点）

Multi AF(MAF)虽然和SAF&&CAF使用相同的调试参数，但是由于MAF是五个Window同时对焦，且已最近焦的Window，因此Performance和行为表现会和SAF及TAF存在一些差异

最终选取最近焦的Window来当做主要的对焦点，这也就是说其中有四个对焦是对在无穷远处，有一个是对焦在30公分左右，这时会以最近焦的点作为主要的对焦点。

  • Continuous AF(CAF)：AF演算法持续对屏幕中心进行监控，当中心场景改变时，自动进行AF。

CAF为了保持对焦过程中画面的Smooth，因此没有使用AF Search Table，而是使用固定的AF Step进行搜索。

2.AF Tuning的介绍

2.1.AF Search Range

决定AF最近及最远的搜寻范围(优先权高于SearchTable)

2.2.AF Search Table

决定sAF，ZSDAF，VCAF的搜寻位置=>Gen AF Table时不同的Step（步数）填入此处

sAF是single AF

ZSD AF是ZSD模式下的AF

VCAF是video模式下的continus AF

2.3.AF Tuning Parameter

AF的调试参数

2.4.Get FV

做完一次AF后，在FV的栏位中显示刚刚的FV

当我们在CCT中按下拍照键，就会做一次AF，这时可以按下Get FV这个按钮，就会在2.2的区域（AF Search Table）中显示刚刚搜寻过程中Gen的FV值，用途是为了更好的适用于特殊场景。

举例

某个场景可能发现在normal Mode里面怎么对焦都对不到，并且AWB偏红黄，这时较为简单的做法就是连接上CCT，在该场景下直接改变Parameter的值，然后重新做AF，生成新的FV值，这时就很容易方便的查看修改的参数产生的影响，复杂的做法就是直接从Code中改变Parameter，然后再BuildSo文件，接着将"so文件"push到机器中，验证修改的参数有没有生效。

2.5.Clear FV

清理上一次AF的FV

4.AF调试Parameter(参数)

4.1.Level1

选择Level1时，可调整Commom的基本Parameter(参数)

解释Commom（公用的）

MT6589的AF的Parameter基本上可以分成三组，这三组分别用在三个不同的Scenario，每一组都有一套AF的Parameter，让使用者根据不同的Scenario去设定不同的Parameter，也就是说可以有三套参数去使用，Commom部分的参数就是独立于这三套的Parameter，属于行为模式的部分，也就是公用的意思。

VAFC_FALL_CNT_DELTA：Video CAF专用，当搜寻方向改变超过此次次数时，停止Video CAF，并将Lens停在最佳Pos。=>当Video CAF搜寻找不到AF时，会来回的搜寻，但是如果次数过多的话，造成用户体验不好，所以这里会设置一个来回搜寻次数的上限值，也就是说就算找不到真正的峰值点，但如果搜寻次数超过设定的上限值，也会停止搜寻的动作。

LV_THRES：Video CAF专用，当环境过暗时，不进行Video CAF，并将Lens停在最佳Pos。=>当环境过暗时，AF的意义不大，并且容易找错，还不如停留在最佳的Position。

SPOT_PERCENT_W：CAF Window位置

SPOT_PERCENT_H：CAF Window位置

InPos：infinity Mode的无穷尽位置

AFC_STEP_SIZE：Video CAF允许的最大一次移动步数。=>如果步数过大，可能会造成移动的时候AF不够Smooth，利用这个限制来阻挡这种不够Smooth的情况发生。

BackJumpPos：当AF找到的Target Pos<BackJumpPos，则分段前往Target Pos(为了避免撞击声)=>为了避免因为惯性而产生的撞击声，当AF往回缩的时候，很有可能因为惯性导致镜头撞击而发出声响

FDWinPercent：为了避免人脸框过大，框到不必要的资讯，因此照此百分比将人脸框內缩。

FDSizeDiff：当人脸的Size改变大于此数值时，会触发CAF重新对焦。

StatGain：HW Output的AF Value，先乘上此数值，才是最后FW使用的AF Value。

4.2.tAF_Coef

选择tAF_Coef时，AF按照下列三组Scenario(情况)区分Parameter

sAF_Coef：SAF Parameter(用于非ZSD的SAF&CAF)

sZSD_AF_Coef：ZSD AF Parameter(用于非ZSD的SAF&CAF)

sVAFC_Coef：Video CAF(用于Video CAF)

THERS_MAIN：判断AF停止搜寻&合焦的主要条件

THERS_SUB：判断AF停止搜寻&合焦的次要条件

INIT_WAIT：为了避免Lens缩回时发出撞击声，将Lens分段移动，最多分为五段(默认值为1)

DONE_WAIT：CAF完成之后，等待多少Frame之后，才Get新的FV当做Reference(默认值是0)

FALL_POS：AF失败时的Lens Default位置(默认值是0)

FRAME_TIME：最短一个Frame的时间(Default是33，用来计算change count的数目)=>当目前曝光时间长，Change count就少，当目前曝光时间短，Change count就多

FIRST_FV_WAIT：第一次进Camera，要等几个Frame才开始Get FV

FV_CHANGE_THRES：FV判断CAF启动的条件(FV变化决定CAF启动)，越低越灵敏(默认是45)

FV_CHANGE_OFFSET：FV判断CAF启动的条件(FV变化决定CAF启动)，越低越灵敏(默认是10000)

FV_CHANGE_CNT：FV判断CAF启动的条件(FV变化决定CAF启动)，越低越灵敏(默认是12)

GS_CHANGE_THRES：亮度判断CAF启动的条件(亮度变化决定CAF启动)，越低越灵敏(默认是0)

GS_CHANGE_OFFSET：亮度判断CAF启动的条件(亮度变化决定CAF启动)，越低越灵敏(默认是20)。

GS_CHANGE_CNT：亮度判断CAF启动的条件(亮度变化决定CAF启动)，越低越灵敏(默认是12)

FV_STABLE_THRES：CAF决定启动之后，Lens正式作动的条件(FV保持稳定，Lens正式作动)，越小，手机需越稳定Lens才作动。

FV_STABLE_OFFSET：CAF决定启动之后，Lens正式作动的条件(FV保持稳定，Lens正式作动)，越小，手机需越稳定Lens才作动。

FV_STABLE_NUM：CAF决定启动之后，Lens正式作动的条件(FV保持稳定，Lens正式作动)，越大，手机需越稳定Lens才作动。

FV_STABLE_CNT：CAF决定启动之后，Lens正式作动的条件(FV保持稳定，Lens正式作动)，越大，手机需越稳定Lens才作动。

在STABLE_NUM+1个frame中，检查是达到标准的次数是否超过STABLE_CNT。

FV_1ST_STABLE_THRES：第一次进入相机之后，CAF第一次启动的条件，Value越小，手机需越稳定CAF才作动。

FV_1ST_STABLE_OFFSET：第一次进入相机之后，CAF第一次启动的条件，Value越小，手机需越稳定CAF才作动。

FV_1ST_STABLE_NUM：第一次进入相机之后，CAF第一次启动的条件，Value越大，手机需越稳定CAF才作动。

FV_1ST_STABLE_CNT：第一次进入相机之后，CAF第一次启动的条件，Value越大，手机需越稳定CAF才作动。

在STABLE_NUM+1个frame中，检查是达到标准的次数是否超过STABLE_CNT。

4.3.SAF

Stable的Parameter,目的是利用OTP中的Calibration Data，对UNIT的AF Table进行微调。

Offset：AF Search Table的起始Position

NormalNum：AF为Normal Mode时，AF Table最大的Index

MacroNum：AF为Macro Mode时，AF Table最大的Index

InfldxOffset：利用OTP的远焦Calibration值，校正AF的Inf Position。

MacroIdxOffset：利用OTP中的近焦Calibration值，校正AF的Macro Position。
