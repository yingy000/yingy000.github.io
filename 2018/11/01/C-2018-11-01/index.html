MT6589_Advance之AF Gen Table

Camera Camera-tuning 2018/11/01

1.名词解释

1.1.Focus Value

Focus Value：表示Contrast Value，用对比值来量化主体的清晰度

1.2.DOF

DOF：表示Deep of Field(景深)，代表物距清晰的范围，景深长，代表物距前后清晰的范围广，景深浅，代表物距前后清晰的范围短。

1.3.Focus Position和Focus Value

Focus Position表示X轴，Focus Value表示Y轴，上图为Focus Value Curve(曲线)的基本原型

从上图可知，零点代表Lens还没有向外推，越往右代表Focus Position推出去的距离越长，推出去的距离越长代表目前离焦点的距离越近，Focus Position推出去的距离越短，代表目前离焦点的距离越远。

2.AF Search Table

2.1.AF Gen Table目的

Gen出一个高品质而且又有效率的AF Table

2.2.AF Search Table原理

利用DOF的概念，控制AF Search Table每一个Search Step之间的间隔小于一定值，确保使用AF Search Table找到的Focus Position，和最佳的Focus Position（Focus Value最大的位置），距离不超过DOF，这样就可以保证主体清晰。

2.3.AF Search Table方法

  • 第一步：找出各个不同距离的Focus Value Curve(曲线)，并求得DOF的大小。

  • 第二步：由于AF在搜寻Max Focus Value Position时，会使用Curve Fitting(曲线拟合)，也就是利用曲线方程式求得最佳解，以提高AF Search的准确度，所以AF Search Table的Step，可以在超过DOF的前提下，仍然保证影像的清晰。

3.Gen AF Search Table的流程

  • 1.利用不同距离的Focus Value Curve，产生DOF Table(a,b,c……j)

一般来说选取3m~10公分的距离来生成AF Table，距离越远可以拍摄的间隔较大，距离较近的话拍摄的间隔就要缩小，举例来说，3m~2m之间的间隔可以拍摄1张或者2张照片，1m~10公分之间的间隔需要拍摄多张照片。

这里的DOF指的是镜头的步数，和平时拍照所说的景深并不相同。

问题：DOF怎么去量化？

假设最清晰点的FB值为100%，只要是任何一个点它的FB值在最佳点的95%以上，就认为这个距离是可以的，拍摄的主体是清晰的。

事实上说只要选取足够高频，一般就可以达到最佳点。

  • 2.利用Curve Fitting方程式，分析在各个距离，满足Focus Position误差<DOF的最大Step(A,B,C…K)

CCT工具会自动估算出最大的Step，从上图中可以看出该Table表最大的Step是A=>原因：当搜寻的点+1，也就是增加一步，它的误差值就会变大，当误差值大于DOF时就不能满足条件，这时回退一步，就找到了最大的Step。

注意：搭配Curve Fitting方程式时，可以提高AF Search的准确度，也就是说如果在某时刻间距大于DOF时，利用Curve Fitting方程式还是可以找到最理想的值，所以Curve Fitting方程式对于寻找Focus Position误差<DOF的最大Step起到十分重要的作用。

问题：为什么需要满足Focus Position误差<DOF所允许的最大Step？

在生成AF Search Table的时候，为了要让AF Search的速度越快，那么两点之间的间隔是越大越好，越大的话速度肯定越快找完全部的Step，可是也不能无止境的往上上升，否则的话最后生成的Focus Position与最佳的Focus Position的误差已经超过DOF，就无法保证拍摄的主体是清晰的。

  • 3.利用满足Focus Position误差<DOF的最大Step(A,B,C…K)，产生全行程的AF Search Table。

首先应该找出一个斜率，确保整个AF Search Table它的每一个搜寻点跟下一个搜寻点之间的距离会小于前面的这个Table（Max allow Step），如果满足这个条件，就可以确保每一个Focus Position去搜寻的Stop最后找出的误差点都会<DOF。

斜率的倾斜程度决定每一个AF Search Table它每一个点的间隔，如果这条线的斜率过高，导致这条线的每一个点没有办法在这之下的话，那么导出来的点就没办法满足error<DOF，只要可以保证这条线的斜率在A~K之下的话，那么导出来的点就可以满足error<DOF，所以需要调整这条线的倾斜程度。

Search Table是指AF在搜寻过程中它的每一次AF的点都是Table上面记录好的点，第一步从0开始找，最大允许的步数为A，所以第二步从A开始找，以此类推……

4.AF Gen Search Table

4.1.工具

CCT AF Page和Camera Enigger Mode（工程模式）

4.2.场景

高对比场景或者是高频直线圆

4.3.距离

3m~5m，其中3m代表无穷远，5m代表极近焦。

4.4.流程

1.进入Engineer Mode，设定好AF及Capture Type

2.距离主体3m开始拍照，一路拍到5cm

3.使用Debug Parser开放照片，开始Gen Table

4.完成

4.5.Start Capture

4.5.1.目的

利用AF Full Search，profiling（分析）出各个距离，Focus Value Curve的特性。

4.5.2.流程

1.进入"Enigger Mode"设定"AF Full Search"

2.从3m开始，每拉近40cm拍一张，距离1m以内，每拉近10cm拍一张，距离10cm以内，每拉近2cm拍一张

3.拍完之后，收集所有图片有Debug Parser工具分析

4.5.3.注意事项

拍摄主体必须为高对比，Edge清楚的图形

手机必须完全固定（三脚架）

光源必须明亮，距离间隔越近，产生出的AF Table准确度越高。

4.6.Analysis Data

4.6.1.目的

利用各个距离Focus value Curve，找出此颗Module景深的特性，并利用此特性建立Focus Search Table

4.6.2.流程

a.利用CCT AF Page开放Full Search AF Picture

b.去除对焦失败的照片(距离越近，影像过糊)，避免错误的Focus资讯影响Table准确度。

c.微调Table Search Point（同样的Focus距离，步数越多，AF越精准，但是速度越慢）
