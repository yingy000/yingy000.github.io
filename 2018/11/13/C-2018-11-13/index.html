工具 CCT 2018/11/13

1.Color Domain

1.1.基本概念

每个颜色（RGB）在HSV Domain上都有对应的坐标，使用HSV Domain调整颜色，原因是因为比较直觉，可分别对色调和饱和度做调整，结果容易预测。

HSV Color Domain坐标轴的物理意义为

  • H：Hue.色调

  • S：Sat.饱和度

  • V：Lum.亮度

注意：调整颜色喜好度，建议只改动Hue和Saturation，使用PCA调整Luminance，容易出现Artifact（方块效应）。

2.3.结论

先利用First Page，框Target File中希望趋近的颜色，ROI中的颜色越单纯越好（比较精准）

得到目标的平均Hue和平均Sat

利用First Page，框Current File中希望改变的颜色，ROI中的颜色一样越单纯越好

得到目前影像的平均Hue和平均Sat

比对目前影像的平均Hue和平均Sat，找出改变的方向

进入Second Page，建立PCA Table，使目前影像的特定颜色趋近目标的特定颜色

3.Gen PCA Table

3.1.利用PCA_GenTable.xlsx产生PCA Table

1.开放CCT Folder中的PCA_GenTable.xlsx

2.在"PCA Table"block改动H(Hue)，S(Sat)的Table

3.产生的PCA Table在"Gen Table"的Block

4.将PCA Table复制下来，贴在新增的*.csv中

5.可使用CCT import此PCA Table，Apply进register或Save to NVRAM

3.2.举例

  • 利用PCA Page1，分析目前色彩和Target的色彩HSV的差异

紫色希望偏蓝一点，饱和度要高一点====>紫色的bin落在20，桃红的bin落在0，得知要趋近Target Color，紫色部分的Hue必须减少

  • 利用PCA_GenTable.xlsx产生PCA Table

改动"PCA Tableblock处的Data

H从bin0~bin20递减，再从bin20~bin39递增（Value>127时为负数，Value’=-(256-Value)）

S从bin0~bin20递增，再从bin20~bin39递减

  • 复制"Gen Table"block

复制10*54的block，并贴入另外一个CVS中

完成PCA Table，如果测试效果不佳，再进行微调

3.3.FW Control

FW中，PCA共有三组Table（HI-D65,MD-CWF,LO-TL84）

Tool中产生PCA Table，一定要选Apply para，Table才会在FW生效

Tool产生的Table若是希望存储起来

  • Export Table：CVS File

  • Save to NVRAM，再Gen Code：CPP

3.4.使用流程

1.产生PCA Table并Import

2.选择色温-a

3.将Table设入FW-b

4.开放PCA-c===>可在Preview及Capture看到改变

5.存储Table-d

  • Export

  • NVRAM
