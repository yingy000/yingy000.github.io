CCT6589_Basic之CDVT Sensor Calibration

1.CDVT Sensor Calibration(Sensor 校准)的了解

1.1.第一部分(Sensor Calibration)(Sensor 校准)

选择测试OB

选择测试Minimum ISO(最小的ISO值)

选择测试Minimum Saturation Gain(最小的饱和Gain值)

1.2.第二部分(Sensor Mode)

选择测试时选择是Preview（预览），Capture或者Video

1.3.第三部分

测试Minimum ISO

1.4.第四部分

测试OB

1.5.第五部分

测试Minimum Saturation Gain

2.测试项目

2.1.OB

2.1.1.测试目的

测试相同Sensor Sensitivity(Sensor 灵敏度)设定下，Sensor运行多次OB测试的平均值，也就是说测试OB是不是稳定的。

2.1.2.测试环境

尽量选择全黑环境，或者直接遮黑镜头。

2.1.3.测试方式

  • 设定Exp Time(曝光时间)

  • 设定Gain值(Sensor Gain)

  • 设定Repeat Times(重复测试的次数)

  • "Run"(开始测试)

  • Result(得到测试结果)

2.2.Minimum ISO

2.2.1.测试目的

测试当Gain值为一倍时(1024)，对应的ISO值应为多少？

也就是说，用来测试Sensor的基本Sensitivity(灵敏度)，和ISO有关。

2.2.2.测试环境

尽量选择适当亮度的均匀光源(DNP灯箱)

2.2.3.测试方式

  • 设定频率(50Hz or 60Hz)

  • 设定LV(根据光源亮度填入数值)

  • 设定F Number(根据光圈大小填入数值)

  • 设定OB(根据2.1的测试结果，填入OB数值)

  • Run(开始测试)

  • Result(得到测试结果)

注意：

测试的结果可以得到"Minimum ISO"，这个值需要填入"Device Info"，Device Info预设会由MTK Camera Tuning Owner提供。

Preview Mode若是Binning Mode，Preview Minimum ISO是Capture Mode的整数倍。

2.3.Minimum Saturation Gain

2.3.1.测试目的

测试能让Sensor饱和(R\G\B Channel均达到255)的最小Gain值

也就是说，当Sensor Gain小于某定值时，Sensor Exposure无论多长，Sensor Output都无法饱和，因此必须找到此Sensor Gain的最小值。

2.3.2.测试环境

尽量选择适当亮度的均匀光源(DNP灯箱)

2.3.3.测试方式

  • 设定频率(50Hz or 60Hz)

  • 设定Target Decline Rate(使用默认)

  • 设定Gain Buffer(Buffer越大，允许的Minimum Saturation Gain越小(使用默认))

  • 设定OB(根据2.1的测试结果，填入OB数值)

  • Run(开始测试)

  • Result(得到测试结果)

注意：测试的结果可以得到"Minimum Saturation Gain"，这个值需要填入"Device Info"，Device Info预设会由MTK Camera Tuning Owner提供。

3. Shading Tuning的了解

3.1.MTK Shading Compensation Solution（MTK Shading的补偿方案）

MTK有三组Shading Table，可以根据需求分别对高中低三种不同色温做补偿

  • MT6575：仅有高/中两组Table可以设定，低色温会套用中色温Table

  • MT6577：高/中/低色温均可使用不同色温的Table

  • MT6589：高/中/低可使用不同色温的Table，另外预留一组Table可以用来补偿特殊色温。

注意：MT6575只开放两种色温的原因：为了避免由于AWB的误判造成shading可能会套用错误的Shading Table，这样会导致画面中的Color shading非常严重。在MT6577中由于技术的升级，可以很正确、清楚的判别，不会导致误判的情况发生，所以可以有三种色温的Shading Table。

3.2.测试环境

  • 准备均匀光源环境(DNP灯箱+滤光片)、拍中性颜色的图片

  • 将手机镜头贴近光源

  • 使用CCT开始Tuning

4.CCT Shading Tuning

4.1.使用步骤

1.选择"Lens Info"

2.Common Control Dialog中，进行Default操作。

在Mode选项中，选择Preview/Capture/Video=>目的在于是选择补偿Preview Shading Table/Capture Shading Table/Video Shading Table

按下"AWB Default"(将设定初始化)=>在Common Control Dialog的右下角，也可以选择手动初始化，但是可能会有遗漏

选择要Calibration(校准)的色温：(高/中/低三组选一组)

MTK的shading Table最多可以区分三组色温：

  • 高色温(6400K)：D75/D65=>使用相同的Shading Table,建议使用DNP补Shading Table

  • 中色温(4500K)：DNP/TL84/CWF=>使用相同的Shading Table，建议使用背光板补Shading Table

  • 低色温：A/Horizon=>使用相同的Shading Table，建议使用背光板补Shading Table

各种色温的Shading Table，直接在要测试的色温环境下Tuning(大光源)，Color shading补偿效果是最好的！但是如果模组不同色温下，两组Shading Table效果差异过大，很有可能会在Switch Shading Table时（切换Shading Table），产生色彩不连续的现象（断层现象），所以应该使用MTK推荐的环境Tuning。

3.取消"Fix Shading"

选择Fix Shading，表示Tuning完一组shading Table，会同时填入NVRAM高中低三组Table。

注意：如果选择Fix Shading之后，可能会导致新的shading Table覆盖旧的shading Table。但是有时有些项目中会需要这种功能。

4.选择Tuning Para

设定Shading要补偿的比例，一般设置为85%~90%。

思考题：为什么补偿的比例不设置为100%？

答：Tuning Para指的是把四周亮度补偿到中心的0.9左右，不补偿到100%的原因是为了更好的适用性。

5.按下Calculate，开始Tuning

6.Tuning完成后将设定存入手机

  • 按下Apply para and Table

  • 按下Apply para

  • 按下Save to NVRAM

重复2、3、4、5步骤，产生三个色温Preview或者Capture的Shading Table

4.2.CCT验证Shading补偿效果

1.由Common Control Dialog Enable Shading

2.指定Shading Table后拍照->高色温的Shading Table/中色温的Shading Table/低色温的Shading Table

3.可以随意点选图片周围的圆圈，显示剖面圆RGB Channel的响应，可以看出Shading的程度。

注意：如果shading补偿完是OK的，那么R、G、B、Y这四条线应该是平直的，紧挨在一起的。

4.3.Import/Export Shading Table

Import：导入文件(CSV格式)

Export：导出文件(CSV格式)

5.重要的文件

5.1.Camera_isp_regs_sensor.h

存放Shading Compensate Register=>存放Sensor ISP的设定

5.2.Camera_isp_lsc_sensor.h

存放Shading Compensate Table=>存放不同种类的Shading Table(Preview/Capture/video)
